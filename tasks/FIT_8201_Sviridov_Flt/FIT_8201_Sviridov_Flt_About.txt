Flt, версия 1.4
Валентин Свиридов, ФИТ, группа 8201.
Программа чтения/записи 24-bit несжатых bmp-изображений
с набором простых фильтров.

Тулбар:
	New - очистить панели
	Load - загрузить изображение из bmp файла
	Save as... - сохранить изображение из зоны C в bmp файл

	Select - выбор части рисунки из зоны A. если изображение слишком
маленькое, то оно сразу перемещается в зону B
	From B to C - перенести изображение из B в C
	From C to B - перенести изображение из C в B

	Grayscale - черно-белое изображение
	Negative - инвертировать цвета

	Floyd-Steinberg dithering - дизеринг по алгоритму Флойда-Стейберга
	Ordered dithering - упорядоченный дизеринг

	Double scale - увеличить изображение в 2 раза

	Roberts operator - определение контуров с помощью оператора Робертса
	Sobel operator - определение контуров с помощью оператора Собеля

	Blur - размытие
	Sharpen - увеличение резкости

	Emboss - тиснение
	Aquarelle - эффект акварели

	Gamma - гамма-коррекция
	Brightness - яркость
	Contrast - контраст

	About - запустить текстовый редактор с этим документом

	Exit - завершить приложение

Все модальные диалоги с выбором настроек для фильтров сохраняют
последнее значение (в том числе гамма, яркость и контраст).

Для гаммы, яркости и контраста используется используемое значение параметра
в 100 раз меньше отображаемого (1 -> 0.01, 1000 -> 10 и шаг соответственно 0.01).

Технические детали по фильтрам:

При получении числа в компоненте цвета < 0, ставится 0.
При получении числа в компоненте цвета > 255, ставится 255.

Grayscale: коэффиценты (0.299, 0.587, 0.114).

Blur: матрица 5x5, с коэффицентом 1/74 (из слайдов),
края шириной 2 пикселя копируются.

Emboss: [[0,1,0],[-1,0,1],[0,-1,0]],
края шириной 1 пиксель копируются.

Sharpen: матрица 3x3,
края шириной 1 пиксель копируются.

Floyd-Steinberg: распределение ошибки на 4 ячейки 
(EAST, SOUTH-EAST, SOUTH, SOUTH-WEST с коэффицентами 
7/16, 1/16, 5/16, 3/16). На краях перераспределения ошибки нет.
То есть если, например, обрабатывается правый край картинки, 
то 7/16 ошибки просто теряется.
Градации строятся с таким расчетом (вообще говоря для общности
можно было бы разрешить выставлять и одну градацию цвета, тогда на 
компоненту шло бы 0 бит, однако из-за вырожденности такую возможность я убрал):
кол-во градаций -> значения
2 -> 0 255
3 -> 0 128 255
4 -> 0 85 170 255 
5 -> 0 64 128 191 255
6 -> 0 51 102 153 204 255 
...

DoubleScale: на краю отсутствующий, но необходимый пиксель
заменяется соседом слева.

Aquarelle: медианный фильтр на 25 точек и Sharpen.
Края шириной 2 пикселя не обрабатываются.

Ordered dithering: неравенство при проверке шаблона и изображения
нестрогое, поэтому белое изображение после обработки фильтром
имеет на 256 точек 1 черную. Замена неравенства на строгое
в свою очередь добаляет белые точки на черное изображение.

Во время выбора части изображения для обработки селектор каждые
200 мсек меняет цвет серый <-> белый.

При необходимости заменой двух констант в коде
размеры панелей делаются произвольными, в том числе
неквадратными, при этом селектор работает корректно.
Вынесено в настройки это не было, посколько отсутствует
соответсвующий use-case для проверки. Тестировать на больших
изображениях однако было приятней.
