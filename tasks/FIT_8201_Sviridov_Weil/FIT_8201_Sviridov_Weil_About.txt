Weil, версия 1.0
Валентин Свиридов, ФИТ, группа 8201.
Программа рисования и построения пересечения многоугольников
на основе алгоритма Вейлера-Азертона.

Что какая кнопка тулбара делает - слева направо:
	New - создать новый документ
	Load - загрузить документ из файла
	Save as... - сохранить файл под определенным именем
	Settings - показать диалог настроек
	Стрелочка #1 (subject) - начать рисование внешней границы области
	Стрелочка #2 (hole) - начать рисовать "дырку" для области
	Стрелочка #3 (clip) - начать рисовать область отсечения
// замечание: направление стрелочек соответсвует предполагаемой ориентации 
// областей  subject, clip -- против часовой стрелки, hole -- по часовой; построить 
// область с неправильной ориентацией не выйдет
	Intersect - пересечь область (возможно, с дыркой) областью отсечения 
		и показать результат
	About - запустить текстовый редактор с открытым этим документом
	Exit - завершить приложение

У мыши обрабатывается событие mousePressed, а не mouseClicked.	 

Как заканчивается построение полигона?
Замыкание по правой кнопке мыши. Нельзя завершить рисование, если точек
меньше 3, есть самопересечения или неправильная ориентация.

По нажатию на Escape можно убрать последнюю нарисованую точку текущего 
полигона. Если точек больше нет -- отменяется рисование полигона.

Строить полигоны по требованиям можно в произвольном порядке.
Правильность ориентации проверяется в момент попытки завершить рисование 
полигона, если она неправильная, то программа попросит это исправить.
По ходу построения проверяется наличие самопересечений - программа не даст
пересечься ребрам.

Если область рисования в файле больше размеров фрейма, то появляются
scroll'ы. При сохранении размер области записывается как размер прямоугольника,
содержащий все полигоны из модели + 50 пикселей.

Используется двойная буферизация и сглаживание.

Суть работы:
Строятся/загружаются полигоны.

Если есть subject и clip, то разрешается получить пересечение
(наличие hole необязательно).

Если hole имеется, то проверяется, что он лежит в subject; иначе
просьба перестроить/убрать.

Проверяются случаи, когда нет пересечений между полигонами.
subject, hole внутри clip, clip внутри hole, clip внутри hole и т.д.

Находятся точки пересечения полигонов и добавляются в списки вершин обоих полигонов
(с ссылками друг на друга).

Находятся точки пересечения, в которых область (subject и hole) входят в clip. 

Пока список входных вершин не пуст,
	создается новый полигон

	из списка берется вершина v, добавляется в полигон, удаляется из списка
	берется следующая за ней v2 в subject/hole и добавляется в полигон
	
	пока не вернёмся в v
	v2 <- следующая за v2, если это не точка пересечения
		следующая за v2 из "другого" списка, если это точка пересечения
		удалить v2 из списка входных точек (если имеется)
	добавить v2 в список точек полигона
	
Таким образом на выходе может быть несколько областей.
